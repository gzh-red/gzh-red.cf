题目描述

农夫约翰希望为他的奶牛们建立一个畜栏。

这些挑剔的畜生要求畜栏必须是正方形的，而且至少要包含C单位的三叶草，来当做它们的下午茶。

畜栏的边缘必须与X，Y轴平行。

约翰的土地里一共包含N单位的三叶草，每单位三叶草位于一个1 x 1的土地区域内，区域位置由其左下角坐标表示，并且区域左下角的X,Y坐标都为整数，范围在$1$到$10000$以内。

多个单位的三叶草可能会位于同一个1 x 1的区域内,因为这个原因，在接下来的输入中，同一个区域坐标可能出现多次。

只有一个区域完全位于修好的畜栏之中，才认为这个区域内的三叶草在畜栏之中。

请你帮约翰计算一下，能包含至少C单位面积三叶草的情况下，畜栏的最小边长是多少。

输入格式
第一行输入两个整数 C 和 N。
接下来 N 行，每行输入两个整数 X 和 Y，代表三叶草所在的区域的X,Y坐标。

同一行数据用空格隔开。

输出格式
输出一个整数，代表畜栏的最小边长。

数据范围
$1 \le C \le 500$,
$C \le N \le 500$

思路
怎么想到用二分?

这道题目显然是要用二分的,因为我们要找到长度最小的栅栏.

用二分怎么做?

我们首先按照题目意思,要我们找到一种栅栏,使它边长最小,所以我们肯定是要二分这个矩阵的长度!

二分判定怎么判定?

可以手动跳转到双指针枚举,但是为了思路一步步升级完善,建议看完下面的,虽然对于解题帮助不是很大.

现在我们已经确定了当前矩阵的长度为mid,但是我们不知道这个矩阵的右上角坐标以及左下角坐标.

温馨提示:

  1. 一个矩阵(正方形是矩阵的一种)可以被唯一确定,当且仅当,你知道它的右上角坐标和左下角坐标
  2. 我们只需要求出右上角坐标$(x,y)$,因为左下角坐标可以通过$(x+len,y+len)$得出,其中len就是上文提到的mid,只不过为了形象理解为正方形长度,所以mid改名为len.

那么我们只需要枚举这个右上角坐标就好了,但是现在我们又面临了一个问题.我们怎么求出这个给定矩阵的三叶草个数

有以下三种做法

  1. 暴力枚举找出矩阵内部有多少株三叶草.单点查询复杂度$O(n^2)$
  2. 二维前缀和,进行预处理有多少株三叶草,总时间复杂度为$O(n^2)$,单点查询复杂度为$O(1)$,空间复杂度$O(10000^{2})$
  3. 双指针判定,只需要枚举点集,不需要枚举右上角坐标,时间复杂度为$O(n^2)$

综上所述,我们只能采用,不需要枚举右上角坐标的双指针判定
是不是觉得上面的长篇大论很废话啊
